## 1. Goal Management System (React + Vite)

A modern wealth advisory CRM and financial goal planner that calculates future needs, inflation impact, SIP step-ups, and current/future gap shortfalls. It allows advisors to track client assets across classes and export professional PDF reports.

### Tech Stack
* **Frontend**: React (v19), Vite (v8)
* **Styling**: Tailwind CSS (v4)
* **Backend**: Custom Node/Express + Prisma API (`server/`) — see "Backend API" below. Replaces the old direct-from-browser Supabase calls and the `localStorage` fallback entirely.
* **Database**: PostgreSQL, accessed only through `server/` via Prisma (`DATABASE_URL`). Currently hosted on Supabase (used purely as a managed Postgres host — no Supabase SDK, RLS, or Supabase-Auth in the data path), chosen to stay **portable**: moving to a self-hosted Postgres later (e.g. a Mac Mini) is a `DATABASE_URL` change, not a rewrite.
* **Auth**: Custom, admin-provisioned (no public signup). bcrypt-hashed passwords, JWT session in an **httpOnly cookie**. Roles: `ADMIN` (+ user management), `MANAGER` (read/write), `VIEWER` (read-only).
* **Libraries**: Lucide React (Icons), Recharts (Visualizations), jsPDF & jspdf-autotable (PDF generation), XLSX (Excel uploads).

### Backend API (`server/`) — Current Data & Auth Layer
* **Stack**: Node (ESM) + Express + Prisma + PostgreSQL. `helmet`, CORS (credentials), `express-rate-limit` on auth routes, `zod` request validation, centralized error handling (`middleware/error.js`).
* **Data model** (`prisma/schema.prisma`): `User`, `Client`, `Goal`, `Mom`, `Task`, `Lead`, `Meeting`, `Prospect`, `AdvisorProfile`. Client/Goal/Mom use real typed columns (camelCase, matching the frontend's object shapes exactly — no snake_case remapping needed, unlike the old Supabase integration). Task/Lead/Meeting/Prospect store the **full object verbatim in a `payload` JSONB column** (a few fields like `stage`/`ownerId`/`groupLeaderId` are additionally promoted to real columns for future indexing).
* **Auth flow**: `POST /api/auth/login` verifies bcrypt hash → signs a JWT → sets it as an httpOnly cookie (never readable by JS). `GET /api/auth/me` re-validates the session on app load. `requireAuth`/`requireRole`/`requireWrite` middleware gate every route; `VIEWER` accounts get a 403 from the server itself on any write, not just a hidden UI button.
* **User provisioning**: `routes/users.js`, admin-only. No signup endpoint exists at all — the seeded admin (`prisma/seed.js`) creates every other account from the in-app **Profile menu → User Management** screen (`UsersAdmin.jsx`).
* **The "cache-over-API" migration pattern** used to move every data module off `localStorage` without touching any component: each frontend store (`services/db.js`, `services/leads.js`, `utils/tasks.js`, `utils/meetings.js`, `utils/prospects.js`, `utils/advisorProfile.js`) keeps its exact original synchronous `loadX()` signature by reading from an in-memory cache. The cache is populated once via a `hydrateX()` call in `App.jsx`'s `loadData()` (fired after login), and `saveX()` updates the cache immediately (so the UI never waits) while persisting to the API in the background — mirroring the "always rewrite the whole array" semantics `localStorage.setItem` used to have (leads/tasks/meetings/prospects sync via a single bulk `PUT` that upserts everything present and deletes anything omitted). This is why none of `LeadsView.jsx`, `TasksView.jsx`, `MeetingsView.jsx`, `BusinessProspects.jsx`, etc. needed any changes — only what's *behind* `loadX()`/`saveX()` changed.
* **Per-user Advisor Profile**: previously a single hardcoded "Nitesh Luthra" profile shared by the whole app (`crm:advisorProfile` in `localStorage`); now `AdvisorProfile` is one-to-one with `User` (`routes/profile.js`, scoped implicitly to `req.user.id` — never a body-supplied id), so every account has its own profile, defaulted from that user's real name/email.

### Core Features & Logic
* **SIP & Goal Calculation Engine (`utils/calc.js`)**:
  * Calculates Future Value (FV) of a target goal incorporating monthly inflation.
  * Projects future value of existing lumpsum investments (`fvOfCurrentInv`) and the recurring SIP stream (`fvOfCurrentSip`) incorporating an annual step-up rate.
  * Solves numerically for the required monthly SIP amount to meet the target using binary search optimization.
  * Derives lumpsum required today to bridge the goal gap.
* **Asset Allocation Net Worth Tracking**:
  * Tracks assets in three categories: Financial, Physical, and Liabilities.
  * Automatically calculates Net Worth (Financial + Physical - Liabilities).
  * Automatically records history of edits and diffs in the database under `asset_allocation.history`.
* **Client Profile & Holdings details**:
  * Captures detailed contact details, internal coverage assignments, family applicants relations (with PAN numbers), and active product holding statuses (Mutual Funds, Term/Medical/Accidental Insurance).
  * Tracks client Status (`Active`, `Dead`, `Inactive`) with badge rendering, and displays 6 new CRM lists (Open Activities, Closed Activities, Meeting Setup History, Business Prospects, Attachments, and Notes).
  * Integrates a premium tabbed layout inside the client form and displays details beautifully in a collapsible dashboard card.
* **Reports**:
  * Timeline-based visual reports of client milestones up to 10 years out.
  * Comprehensive PDF exporter (`utils/pdf.js`) compiling summary tiles, tables, and projections.
